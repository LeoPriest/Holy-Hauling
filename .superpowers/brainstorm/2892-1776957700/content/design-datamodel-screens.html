New: User model (SQLite table)

-- users table
id VARCHAR PRIMARY KEY (uuid)
username VARCHAR NOT NULL UNIQUE
credential_hash VARCHAR NOT NULL (bcrypt — PIN today, password later)
role VARCHAR NOT NULL (admin | facilitator | supervisor | crew)
is_active BOOLEAN DEFAULT true
created_at DATETIME NOT NULL
created_by VARCHAR (user_id of admin who created)
Seeded at startup: one default admin user with PIN 0000 if the table is empty. First thing admin does is change it.

New API routes

POST /auth/login — body: {username, pin} → returns {token, user}. No auth required.
GET /auth/me — returns current user from token. Used on app load to verify token still valid.
GET /admin/users — list all users. Admin only.
POST /admin/users — create user. Admin only.
PATCH /admin/users/{id} — update role, reset PIN, deactivate. Admin only.
GET /jobs — booked leads, shaped for crew/supervisor. Auth required, any role.
PATCH /jobs/{id}/status — en_route / started / completed. Supervisor only.

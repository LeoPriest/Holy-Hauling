4 Roles

admin
Full access. User management, all settings, revenue history, pricing (later). Only role that can create/deactivate users.
facilitator
Current app: lead queue, intake, AI review, notes, settings (read+write). No user management.
supervisor
Job list (booked leads). Full job detail incl. customer phone + quote. Add photos. Mark en route / started / complete.
crew
Job list only. Address, size, stairs, scope. Add photos. No phone, no quote, no mark-complete.

Auth Flow

Login: User types name → taps 4-digit PIN keypad → POST /auth/login → server returns a signed JWT containing { user_id, username, role } → stored in localStorage
Every request: Frontend attaches Authorization: Bearer <token> header → FastAPI middleware decodes + validates → injects current_user → route uses it for role checks and audit trail
Token life: 30-day expiry. Auto-logout on expiry. Admin can deactivate a user (all subsequent requests rejected even with a valid token).
PIN storage: Hashed with bcrypt. Field named credential_hash so upgrading to a full password later = UI change only, no schema migration.

What changes in the existing app

✅ All existing routes get a current_user = Depends(require_auth) — no role restriction, just must be logged in
🔒 /settings PATCH and DELETE /leads get require_role("admin", "facilitator")
🔒 /admin/users/* gets require_role("admin") — new routes
📱 Frontend reads role from decoded JWT → shows/hides nav items → redirects crew to /jobs instead of /
🔑 Login screen shown when no valid token in localStorage

# Holy Hauling – Master Lead System (v2.2 FULL)
**Flowcharts • Scripts • Gates • Pricing Lenses • Follow-Up Rules • Moving + Hauling**

This master document merges the original Comprehensive Lead System Playbook with the v2.1 refinements (Gate 2A/2B split, pricing lenses, flat-rate eligibility checks, and follow-up discipline). Use this as the **single source of truth** for handling Thumbtack-first leads from first response to job tracking.

---

## 1. System Philosophy *(Locked)*

> The goal is not to maximize booked jobs. The goal is to maximize **profitable, controlled work with repeatable execution**. Releasing misaligned leads is a successful outcome.

- Respond fast enough to compete, but never sacrifice control to win a lead.
- Delay pricing until engagement + minimum clarity is achieved (unless using a Gate 2A alignment range).
- Price based on the correct lens: **Moving = time + care; Hauling = volume + disposal.**
- Flat rates are **earned** (after clarity), not guessed (before clarity).

## 2. End-to-End System Map

| Stage | Gate | Primary Goal | Output / Next Step |
|-------|------|-------------|-------------------|
| **Lead Arrival** | Gate 0 | Control effort before investing time | Full Reply / Minimal Reply / Release |
| **Message Flow** | Gate 1 | Get reply + confirm timing & baseline scope | Proceed or follow-up sequence |
| **Alignment** | Gate 2A | Confirm expectation alignment (ranges/guardrails) | Aligned / Misaligned |
| **Commitment** | Gate 2B | Lock price + schedule with clear terms | Booked / Released |
| **Execution** | — | Deliver job + collect payment | Job Tracker + review/referral ask |

**System rule of thumb:**
> If you are not yet aligned on timing + scope (Gate 1), do not lock a price (Gate 2B).
> If you are not aligned on expectations (Gate 2A), do not chase.

## 3. Gate 0 — Pre-Engagement Triage *(Moving + Hauling)*

Gate 0 happens immediately when a lead arrives. It determines effort level before you type a message.

| Signal | Why It Matters | Gate 0 Output |
|--------|---------------|---------------|
| Small job + long distance | Travel erases margin | Minimal / Release |
| Same-day + vague scope | High risk + low control | Minimal |
| Cheapest-only language | High friction; low close rate | Release |
| High lead cost vs. small job | Negative ROI risk | Minimal / Release |
| Whole-house haul-out / "4+ truckloads" | High variance + dump fees | Full reply with controls |

### Effort Outputs *(Locked)*
- **Full Reply** — Run full Gate 1 message flow.
- **Minimal Reply** — Send Message 1 only (and at most Message 3). No chasing.
- **Release** — Polite decline/close loop (or no response if appropriate).

---

## 4. Gate 1 — Thumbtack Message Flow *(Locked)*

Gate 1 exists to get a reply and confirm timing + baseline scope. Selling and detailed pricing are delayed.

```
New Lead Arrives
    → Message 1 – Clarify / Confirm timing (date-aware)
        Reply? Yes → Message 2 (2 quick details) → Gate 2A
        Reply? No  → Message 3 (timing-aware re-engagement)
            Reply? Yes → Gate 2A
            Reply? No  → Message 4 (next morning)
                Reply? Yes → Gate 2A
                Reply? No  → Message 5 (close loop + release)
```

## 5. Thumbtack Scripts *(Locked — Copy/Paste)*

### Message 1 — Clarify / Confirm *(choose matching variant)*

- **Specific date listed:**
  > *"Hey [Name] — I see you're looking at [Day/Date]. Just confirming: are you flexible on timing, or does it need to be that day?"*

- **Multiple dates listed:**
  > *"Hey [Name] — I see a few dates listed ([Date A] / [Date B]). Which one matters most for you?"*

- **ASAP:**
  > *"Hey [Name] — I see you're looking for ASAP. Is this today/tomorrow, or later this week?"*

### Message 2 — Minimal Details *(2 questions max)*
- **Moving:** *"Perfect. About how many large items (couch/bed/dresser), and any stairs, tight turns, or long carry?"*
- **Hauling:** *"About how much total volume are we talking (single item vs pile vs multiple loads), and any mattresses/paint/tires/electronics?"*

### Message 3 — Timing-Aware Re-Engagement
- *"Just checking in — are you leaning toward [Date A] or [Date B]?"*
- *"Just checking in — are you trying to get this done in the next couple days, or more flexible?"*

### Message 4 — Next Morning
> *"Morning — I can confirm price and timing whenever you're ready. Roughly how many large items / how much volume, and any stairs or tight access?"*

### Message 5 — Close Loop
> *"No worries if you already found help — do you want me to close this out, or are you still needing assistance?"*

## 6. Gate 2A — Alignment Test *(Locked)*

Gate 2A checks expectation alignment before any firm price is locked. Use ranges, guardrails, and the correct pricing lens to test alignment. Silence after Gate 2A is a valid "misaligned" signal.

| Service | Alignment Tool | What You Are Testing |
|---------|---------------|----------------------|
| **Moving** | Process + timing + scope clarity | Seriousness + readiness to schedule |
| **Hauling** | Volume-based ranges (tiered) | Budget alignment for high-volume jobs |

### Gate 2A Rules *(Locked)*
- Gate 2A may include a range or tiered estimate — it is **not** a locked price.
- One follow-up is allowed after Gate 2A. **Never adjust pricing on the follow-up.**
- If the lead is misaligned, release quickly and professionally.

---

## 7. Gate 2B — Commitment Test *(Lock Price + Schedule)*

Only leads that pass Gate 2A enter Gate 2B. This is where you lock a flat rate or hourly structure, confirm timing, and commit resources.

### Moving — Preferred Structures
- **Hourly + minimum** — best when control variables are uncertain
- **Flat rate with guardrail** — best when control variables are confirmed
- **Flat rate + wait-time clause** — best when elevator/truck/parking delays are possible

### Hauling — Preferred Structures
- **Tiered volume pricing** (by load / fraction of truck) for small-to-medium jobs
- **High-volume jobs:** tiered range at Gate 2A → firm number at Gate 2B after final clarifications
- **Dump fees:** separate (recommended) or included up to a cap with a pause-and-confirm rule

## 8. Pricing Lenses + Flat-Rate Eligibility *(Lessons Learned)*

| Service | Pricing Lens | Primary Risks to Control |
|---------|-------------|--------------------------|
| **Moving** | Time + Care | Access, stairs, prep, elevator reality, motorized items, delays |
| **Hauling** | Volume + Disposal | Hidden volume, dump fees, heavy items, multiple trips, fatigue |

### Flat-Rate Moving — Only Allowed When ALL Are Confirmed

| Control Check | Required |
|---------------|----------|
| True freight elevator (not passenger-style) | ✅ Yes |
| No motorized/adjustable heavy items — OR quoted with premium/guardrail | ✅ Yes |
| Wrapping completed — or wrapping clause added | ✅ Yes |
| Holy Hauling controls the truck timing — or wait-time clause added | ✅ Yes |

### Key Clarification Questions to Prevent Underbids
- *"Freight elevator" → Is it a true freight elevator (large cab) or passenger-style?*
- *Any motorized items (adjustable bed, treadmills, recliners)?*
- *Is furniture already wrapped, or will you want us to wrap on-site?*
- *Who is providing the truck (us or customer), and when will it arrive?*

## 9. Lead Intake System — What to Capture + AI Handoff

Capture only what you need to price and schedule confidently.

| Field | Moving | Hauling |
|-------|--------|---------|
| **Timing** | Date/time window confirmed | Date/time window confirmed |
| **Scope baseline** | Large items + stairs/access | Volume estimate + key items |
| **Risk flags** | Elevator reality, motorized, wrapping, truck control | Mattresses/paint/tires/e-waste; heavy items; dump runs |
| **Pricing model** | Hourly vs. flat w/ guardrail | Tiered volume + disposal |
| **Status** | Gate 0/1/2A/2B | Gate 0/1/2A/2B |

### AI Lead Facilitator — Recommended Use
- Use AI **after Gate 1 engagement** when you have timing + baseline scope.
- Use AI heavily for **moving**: structuring quotes, options, pricing explanation, and close-ready messages.
- For **hauling**: AI can help draft tiered ranges and close language, but pricing remains volume/disposal driven.

## 10. Pricing Explanation Templates

### Moving — Neutral "How Much?"
> *"Based on what you shared, you're likely looking at $___–$___. That includes labor, travel, loading/unloading, and protection (blankets/straps). I can lock it in exactly with one or two details, or we can keep it simple here — your call."*

### Moving — "That Seems Expensive"
> *"Totally fair — I want you to feel good about it. We price to cover the real labor time, travel, and protection so the job doesn't run long or turn into surprise fees. If it helps, I can give you two options: basic or full-service. Which matters more today — lowest price, or most hands-off?"*

### Hauling — Volume Tier Anchor *(Gate 2A)*
> *"For haul-outs, pricing is based on total volume and disposal. As a reference: up to ~4 full pickup loads is typically $___–$___; 5+ loads is usually $___–$___+. Dump fees are additional and based on weight."*

## 11. Follow-Up Discipline + Release Scripts

### One Follow-Up Rule *(Locked)*
- After Gate 2A, send **at most one follow-up** ~24 hours later.
- **Never adjust pricing** in the follow-up.
- If still no response, release and move on.

### Follow-Up Script *(Gate 2A)*
> *"Hi [Name] — just circling back in case my last message got buried. If the range/process I shared works for you, I'm happy to help. If you already made other arrangements, no worries at all — just wanted to check in."*

### Release Script *(Polite Close)*
> *"Appreciate you reaching out — based on what you're looking for, I may not be the best fit. I don't want to overpromise. Wishing you the best getting it handled."*

## 12. Transition to Job Tracker + Post-Job Process

Once a job is booked (Gate 2B), the lead system ends and execution begins.

1. Create Job ID (if using the ID generator)
2. Move confirmed details into the **Job Tracker**: date/time, addresses, scope summary, price, payment method, crew, notes
3. **Day-of:** Reconfirm arrival window and parking/loading instructions
4. **After completion:** Request review/referral while satisfaction is highest
5. If tip/add-on is promised via Zelle and it's first-time: confirm it lands before you leave or offer Venmo/Cash App fallback
